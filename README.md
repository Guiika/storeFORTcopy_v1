# Веб-приложение магазина

Проект представляет собой веб-приложение интернет-магазина с реализованной регистрацией пользователей, каталогом товаров, категориями, корзиной, избранным, профилем пользователя и административной панелью.

Backend (готов) уже содержит основную бизнес-логику и API для работы с товарами, пользователями, категориями, корзиной и избранным. Frontend (в процессе 🥹🥹🥹🥹) использует готовые API-методы и отображает пользовательский интерфейс магазина почти.

---

## Реализованный функционал

### 1. Регистрация и авторизация

Реализована работа с пользователями:

- регистрация нового пользователя;
- вход пользователя в систему;
- авторизация по email и паролю;
- хранение основных данных пользователя.

При регистрации используются следующие данные:

- email;
- пароль;
- имя;
- фамилия;
- телефон;
- адрес.

Для входа используются:

- email;
- пароль.

---

### 2. Пользователь

В системе предусмотрена сущность пользователя.

Основные данные пользователя:

- id;
- email;
- имя;
- фамилия;
- телефон;
- адрес;
- дата создания аккаунта.

Также реализованы функции для работы с профилем пользователя:

- получение данных профиля;
- обновление профиля;
- смена email;
- просмотр статистики пользователя.

---

### 3. Категории товаров

Реализована работа с категориями товаров.

Доступный функционал:

- получение всех категорий;
- создание новой категории;
- обновление категории;
- проверка ошибки при создании категории, если такая категория уже существует.

Для категории могут использоваться следующие данные:

- название;
- описание;
- `parent_id` для связи с родительской категорией.

Это позволяет реализовать структуру категорий и подкатегорий.

---

### 4. Товары

Реализована работа с товарами магазина.

Доступный функционал:

- создание товара;
- получение списка всех товаров;
- обновление данных товара;
- загрузка изображений товара;
- подсчёт общего количества товаров;
- проверка наличия товаров;
- проверка остатков на складе;
- счётчик просмотров товаров;
- получение новых поступлений;
- получение товаров со скидкой.

Для товара могут использоваться следующие данные:

- название;
- описание;
- цена;
- старая цена;
- id категории;
- количество;
- цвет;
- размер;
- материал;
- бренд;
- изображения.

Товары со скидкой определяются по условию:

```text
old_price > price
```

---

### 5. Каталог

Реализован каталог товаров.

Доступный функционал:

- быстрый поиск товаров;
- фильтрация товаров;
- отображение списка товаров;
- работа с категориями и подкатегориями;
- поиск по параметрам товара.

Каталог позволяет пользователю находить нужные товары по названию, категории и другим характеристикам.

---

### 6. Избранное

Реализован функционал избранных товаров.

Доступный функционал:

- добавление товара в избранное;
- получение списка избранных товаров;
- удаление товара из избранного;
- очистка избранного;
- получение статистики избранного;
- получение популярных товаров на основе избранного;
- отображение только товаров, которые есть в наличии.

Избранное позволяет пользователю сохранять интересующие товары и позже возвращаться к ним.

---

### 7. Корзина покупок

Реализована корзина покупок.

Доступный функционал:

- добавление товара в корзину;
- получение корзины пользователя;
- подсчёт количества товаров в корзине;
- проверка наличия товара в корзине;
- проверка наличия товара на складе;
- перемещение товара из избранного в корзину;
- удаление товара из корзины;
- применение промокода, если он предусмотрен логикой проекта.

Корзина используется для подготовки товаров к оформлению заказа.

---

### 8. Профиль пользователя

Реализована страница профиля пользователя.

Доступный функционал:

- отображение данных пользователя;
- обновление профиля;
- просмотр статистики пользователя;
- смена email.

Профиль нужен для управления личной информацией пользователя.

---

### 9. Административная панель

Реализована административная часть проекта.

Доступный функционал администратора:

- админ-панель;
- просмотр всех пользователей;
- изменение роли пользователя;
- блокировка пользователя;
- массовое обновление цен;
- экспорт товаров в CSV;
- просмотр системной информации;
- просмотр статистики магазина.

Административная панель предназначена для управления пользователями, товарами и общей информацией магазина.

---

### 10. Статистика и dashboard

Реализованы данные для административной статистики.

В dashboard могут отображаться:

- количество зарегистрированных пользователей;
- количество активных товаров;
- количество активных категорий;
- количество заказов;
- популярные товары;
- товары, которые скоро закончатся на складе.

---

## Роли пользователей

В проекте предусмотрено разделение пользователей по ролям.

Основные роли:

- обычный пользователь;
- администратор.

Обычный пользователь может:

- регистрироваться и входить в аккаунт;
- просматривать каталог;
- искать и фильтровать товары;
- добавлять товары в корзину;
- добавлять товары в избранное;
- управлять своим профилем.

Администратор может:

- управлять категориями;
- управлять товарами;
- просматривать пользователей;
- изменять роли пользователей;
- блокировать пользователей;
- обновлять цены;
- экспортировать товары;
- просматривать системную информацию.

## Основные сущности проекта

### User

Пользователь системы.

Поля:

```text
id
email
password
first_name
last_name
phone
address
created_at
role
```

---

### Category

Категория товара.

Поля:

```text
id
name
description
parent_id
```

---

### Product

Товар магазина.

Поля:

```text
id
name
description
price
old_price
category_id
quantity
color
size
material
brand
images
```

---

### Cart

Корзина пользователя.

Используется для хранения товаров, которые пользователь планирует купить.

Функции:

- добавление товара;
- удаление товара;
- получение корзины;
- подсчёт количества товаров;
- проверка товара в корзине.

---

### Wishlist

Избранное пользователя.

Используется для хранения товаров, которые пользователь отметил как понравившиеся.

Функции:

- добавление товара;
- удаление товара;
- получение избранного;
- очистка избранного;
- статистика избранного.

## Краткое описание проекта

Это интернет-магазин с пользовательской и административной частью.  
Пользователь может зарегистрироваться, просматривать товары, искать их в каталоге, добавлять в корзину и избранное, а также управлять своим профилем.  
Администратор может управлять товарами, категориями, пользователями, ценами и получать статистику по магазину.
